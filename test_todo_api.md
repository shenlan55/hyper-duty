# 待办事项API测试用例

## 测试环境
- 前端：http://localhost:5174
- 后端：Spring Boot 服务运行中
- 浏览器：Chrome

## 测试用例

### 1. 加班审批API - 正常场景（正例）
**测试步骤**：
1. 启动前端应用和后端服务
2. 登录系统
3. 访问首页
4. 检查待办事项区域是否显示
5. 打开浏览器控制台
6. 观察加班审批API请求是否成功

**预期结果**：
- 浏览器控制台显示GET请求到 `/duty/record/pending/{employeeId}`
- 请求返回状态码200
- 响应数据为数组格式
- 待办事项区域显示加班审批待办
