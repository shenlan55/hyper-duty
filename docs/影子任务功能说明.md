# 影子任务功能说明

## 功能概述

影子任务功能允许用户从一个项目的任务创建一个"影子"到另一个项目中，实现跨项目的任务参考和协作。

## 核心特性

### 1. 影子任务创建
- 从任意项目选择源任务
- 为影子任务自定义名称和描述
- 将影子任务关联到目标项目

### 2. 影子任务查看
- 实时同步源任务的状态、进度等信息
- 显示源项目和源任务信息
- 查看影子任务的批注列表

### 3. 批注功能
- 为影子任务添加批注
- 支持删除批注
- 显示批注人和时间

### 4. 源任务查看影子
- 在源任务详情页面查看所有影子
- 查看每个影子的目标项目和批注数

## 数据库表结构

### pm_task_shadow（影子任务表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGSERIAL | 主键 |
| source_task_id | BIGINT | 源任务ID |
| source_project_id | BIGINT | 源项目ID |
| target_project_id | BIGINT | 目标项目ID |
| target_task_id | BIGINT | 目标任务ID（预留） |
| shadow_name | VARCHAR(200) | 影子任务名称 |
| shadow_description | TEXT | 影子任务描述 |
| create_by | BIGINT | 创建人ID |
| create_time | TIMESTAMP | 创建时间 |
| update_time | TIMESTAMP | 更新时间 |
| is_deleted | SMALLINT | 是否删除 |

### pm_shadow_annotation（影子任务批注表）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGSERIAL | 主键 |
| shadow_id | BIGINT | 影子任务ID |
| content | TEXT | 批注内容 |
| employee_id | BIGINT | 批注人ID |
| employee_name | VARCHAR(100) | 批注人姓名 |
| attachments | TEXT | 附件JSON |
| create_time | TIMESTAMP | 创建时间 |
| update_time | TIMESTAMP | 更新时间 |
| is_deleted | SMALLINT | 是否删除 |

## 后端API

### 影子任务接口
- `POST /pm/shadow` - 创建影子任务
- `DELETE /pm/shadow/{id}` - 删除影子任务
- `GET /pm/shadow/{id}` - 获取影子任务详情
- `GET /pm/shadow/target-project/{targetProjectId}` - 获取项目的影子任务列表
- `GET /pm/shadow/source-task/{sourceTaskId}` - 获取源任务的影子列表
- `POST /pm/shadow/query` - 查询影子任务

### 批注接口
- `POST /pm/shadow/annotation` - 添加批注
- `DELETE /pm/shadow/annotation/{id}` - 删除批注
- `GET /pm/shadow/annotation/shadow/{shadowId}` - 获取影子任务的批注列表

## 使用流程

### 创建影子任务
1. 进入任务详情页面或项目详情页面
2. 点击"创建影子"或"新建影子"按钮
3. 在对话框中选择源任务
4. 选择目标项目
5. 可选：自定义影子名称和描述
6. 点击"创建"完成

### 查看影子任务
1. 进入目标项目的详情页面
2. 在"影子任务"卡片中查看列表
3. 点击某个影子任务查看详情
4. 在详情中可以查看源任务信息和添加批注

### 查看源任务的影子
1. 进入源任务的详情页面
2. 在"影子任务"区域查看所有影子
3. 可以点击查看每个影子的详情

### 添加批注
1. 打开影子任务详情
2. 点击"添加批注"按钮
3. 输入批注内容
4. 点击"提交"保存

## 注意事项

- 影子任务本身不存储任务状态和进度，这些信息实时从源任务获取
- 删除影子任务不会影响源任务
- 影子任务的批注只属于该影子，与源任务无关
