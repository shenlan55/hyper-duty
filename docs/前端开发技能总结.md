# 前端开发技能总结

## 📚 核心技能

### 1. Vue 3 开发技巧

**响应式数据管理**
- **ref vs computed**：理解何时使用ref（存储可变值）和computed（基于其他响应式数据计算值）
- **模板自动解包**：在Vue 3模板中，ref对象会自动解包，无需使用`.value`访问值
- **reactive vs ref**：对于复杂对象使用reactive，对于基本类型或需要重新赋值的对象使用ref

**组件开发**
- **Element Plus 组件库**：熟练使用Element Plus组件，包括表单、选择器、日期选择器等
- **组件通信**：掌握props、emit、provide/inject等组件通信方式
- **表单验证**：使用Element Plus的表单验证功能，确保用户输入的正确性

### 2. JWT 令牌处理

**令牌解码**
- **Base64解码**：掌握JWT令牌的Base64解码方法，获取用户信息
- **错误处理**：处理令牌解码失败的情况，确保页面不会崩溃
- **信息提取**：从解码后的令牌中提取用户信息，如用户名、员工ID等

**令牌存储与管理**
- **localStorage**：使用localStorage存储令牌和用户信息
- **状态管理**：结合Pinia管理用户认证状态
- **令牌过期处理**：处理令牌过期的情况，及时跳转到登录页面

### 3. 错误分析与排查

**常见错误类型**
- **ReferenceError**：变量未定义，通常是因为变量声明在使用之后
- **TypeError**：类型错误，如访问undefined的属性
- **403 Forbidden**：服务器拒绝访问，可能原因包括：
  - 令牌过期或无效
  - 前后端传递参数格式不一致
  - 权限不足
  - 请求头缺少必要信息

**错误排查技巧**
- **浏览器控制台**：使用浏览器控制台查看错误信息和堆栈跟踪
- **网络请求分析**：使用网络面板分析请求和响应，检查参数格式和状态码
- **代码审查**：检查代码中的变量声明顺序、类型转换、异步操作等
- **日志记录**：在关键位置添加日志，方便排查问题

### 4. 前端性能优化

**代码优化**
- **减少重复代码**：抽取公共函数和组件，避免代码重复
- **按需加载**：使用动态导入（import()）实现代码分割，减少初始加载时间
- **计算属性缓存**：使用computed属性缓存计算结果，避免重复计算

**资源优化**
- **图片压缩**：压缩图片资源，减少加载时间
- **CSS优化**：合并CSS文件，减少HTTP请求
- **JS优化**：压缩JS文件，移除未使用的代码

### 5. 前后端交互

**API调用**
- **Axios**：使用Axios发送HTTP请求，处理响应和错误
- **请求拦截器**：在请求中添加令牌和其他必要信息
- **响应拦截器**：统一处理响应和错误，如401（未认证）、403（禁止访问）等

**参数传递**
- **请求体格式**：确保请求体格式与后端API要求一致
- **查询参数**：正确传递查询参数，处理特殊字符
- **路径参数**：使用路径参数传递ID等信息

### 5. 布局设计与响应式适配（2026-06-24 新增）

**左右分栏布局实现**：
```css
/* 外层容器 */
.layout-container {
  display: flex;
  gap: 16px;
  align-items: stretch; /* 关键：让左右两侧等高同步 */
}

/* 左侧固定宽度面板 */
.left-panel {
  width: 280px;
  min-width: 280px;
  flex-shrink: 0;
}

/* 右侧自适应面板 */
.right-panel {
  flex: 1;
  min-width: 0; /* 防止flex子项溢出 */
}
```

**Flexbox 等高同步技巧**：
- 使用 `align-items: stretch` 让左右两侧自动等高
- 避免使用固定 `height`，改用 `flex: 1` 让内容自适应填充
- 卡片内部使用 `display: flex; flex-direction: column` 实现垂直布局

**树形组件 + 表格组合场景**：
- **左侧树**：展示层级结构（如部门、分类），支持搜索、点击筛选
- **右侧表格**：展示详细数据列表，支持分页、搜索、CRUD
- **数据联动**：通过状态变量（如 `selectedId`）控制右侧数据源

**避免常见问题**：
1. **底部空白**：树组件设置 `flex: 1` 自动填充空间
2. **不同步缩放**：外层容器不要设置固定高度，让内容自然撑开
3. **移动端适配**：使用媒体查询（@media）在小屏幕时改为上下堆叠
4. **内容溢出**：右侧面板添加 `min-width: 0` 和 `overflow` 处理

**实战案例 - 人员管理页面改造**：
- 改造前：单栏表格 + 下拉框筛选部门
- 改造后：左侧部门树（280px）+ 右侧人员表格（自适应）
- 效果：组织架构可视化、快速定位、人数统计一目了然

## 🔍 问题分析案例

### 案例1：403 Forbidden错误

**症状**：前端发送请求时，服务器返回403 Forbidden错误

**可能原因**：
1. **令牌问题**：JWT令牌过期或无效
2. **参数格式**：前后端传递参数格式不一致
3. **权限问题**：用户没有访问该资源的权限
4. **请求头**：请求头缺少必要信息

**排查步骤**：
1. 检查浏览器控制台的网络请求，查看请求头和参数
2. 验证JWT令牌是否有效
3. 检查后端API文档，确认参数格式
4. 联系后端开发人员，确认权限设置

### 案例2：TypeError: Cannot read properties of undefined

**症状**：浏览器控制台报错"TypeError: Cannot read properties of undefined"

**可能原因**：
1. **变量未定义**：访问了未定义的变量或对象属性
2. **异步操作**：在异步操作完成前访问了结果
3. **条件判断**：缺少对空值的条件判断

**排查步骤**：
1. 检查报错位置的代码，确认变量是否定义
2. 检查异步操作的处理方式，确保在结果返回后再访问
3. 添加对空值的条件判断，避免访问undefined的属性

### 案例3：Vue事件处理与防抖

**症状**：登录页面验证码弹框按回车键关闭，输入密码后按回车重复触发两次验证码

**可能原因**：
1. **事件冒泡**：回车键事件同时在多个元素上触发（el-form、el-input）
2. **缺少防抖**：没有防止重复提交的机制
3. **Element Plus默认行为**：el-dialog和el-form有默认的表单提交行为

**解决方案**：
1. **使用事件修饰符**：
   - `@keyup.enter.prevent.stop`：阻止默认行为 + 阻止冒泡
   - `@keydown.enter.prevent.stop`：在keydown阶段就拦截事件
   - `@submit.prevent`：阻止表单默认提交

2. **添加防抖机制**：
   - 使用状态变量（`isSubmitting`、`isVerifying`）防止重复请求
   - 在 `try-finally` 中确保状态变量被正确重置

3. **代码示例**：
   ```javascript
   const isSubmitting = ref(false)
   
   const handleSubmit = async () => {
     if (isSubmitting.value) return
     try {
       isSubmitting.value = true
       // 执行逻辑
     } finally {
       isSubmitting.value = false
     }
   }
   ```

4. **模板示例**：
   ```vue
   <el-dialog @keyup.enter.prevent.stop>
     <el-form @submit.prevent @keyup.enter.prevent.stop>
       <el-input @keydown.enter.prevent.stop @keyup.enter.prevent.stop />
     </el-form>
   </el-dialog>
   ```

### 6. 响应式设计与移动端适配

**双模布局策略**
- 使用 CSS 媒体查询 `@media (max-width: 767px)` 隔离 PC 和移动端样式
- `Layout.vue` 双模布局主组件：PC 侧边栏 <-> 移动端底部导航 + 汉堡菜单抽屉
- 移动端通过 `useResponsive` Hook 检测设备类型（`isMobile` / `isTablet`）

**核心组件**
- `MobileBottomNav.vue`：底部 Tab 导航，4 个入口（首页、值班、任务、待办），图标+文字组合，28px 大图标适配触摸
- `MobileDrawer.vue`：汉堡菜单抽屉，递归渲染完整菜单树，支持展开/收起子菜单
- `MobileGuard.vue`：B 类页面移动端拦截，提示用户切换到 PC 访问
- `BaseTable.vue`：移动端自动切换卡片模式，树形数据支持 **▶/▼ 箭头**递归展开收起子任务

**样式隔离规范**
- `mobile.scss`：移动端全局样式覆盖（仅 `@media` 内生效），不影响 PC
- 按钮对齐：全局 `el-button + .el-button { margin-top: 8px }` 适配纵向排列，弹性容器内通过 10+ 个选择器重置 `margin-top: 0`
- 触摸优化：按钮最小高度 44px，底部导航适配 iPhone 安全区域

**常见移动端问题**
- **按钮错位**：弹性容器中的按钮受全局 `margin-top` 影响 → 在 `.el-dialog__footer`、`.el-form-item__content`、`[style*="display: flex"]` 等容器重置
- **操作列溢出**：表格操作列按钮过多时换行错乱 → 操作列禁用换行（`white-space: nowrap`），卡片模式底部 flex 布局
- **下拉框过宽**：`el-select` 和 `el-date-picker` 移动端宽度坍缩 → 全局设置 `width: 100%`
- **分页器控件多**：移动端分页器简化 → 隐藏页码跳转、总条数，仅保留上一页/下一页
- **树形数据不显示子节点**：卡片模式直接遍历顶层数组 → `flatTreeData` 递归扁平化，根据 `expandedRows` 动态展开收起子任务
- **按钮尺寸过大**：移动端表格/卡片按钮过大导致溢出 → 在 `mobile.scss` 深度优化按钮尺寸：
  - 表格按钮：高度从 32px 缩小到 **22px**，字体从 14px 缩小到 **10px**
  - 卡片按钮：同样缩小到 **22px** 高度，padding 最小化
  - 按钮间距：缩小到 **2-4px**，添加 `flex-wrap: wrap` 支持换行
  - 操作列单元格：padding 进一步减小，gap 最小化
