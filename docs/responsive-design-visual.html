三、技术方案详解

3.1 响应式断点策略

// styles/breakpoints.scss (新建) $breakpoint-sm: 768px; // 手机/平板分界 $breakpoint-md: 1024px; // 平板/PC分界 $breakpoint-lg: 1440px; // 大屏PC @mixin mobile { @media (max-width: $breakpoint-sm) { @content; } } @mixin tablet { @media (min-width: $breakpoint-sm + 1) and (max-width: $breakpoint-md) { @content; } }

3.2 Layout.vue 改造核心逻辑

// 双模式布局切换 const isMobile = ref(window.innerWidth < 768) // PC端：侧边栏 + 标签页 + 内容区（现有逻辑保持不变） // 手机端： // 1. 侧边栏 → 隐藏，改为底部Tab栏 + 汉堡抽屉 // 2. 标签页 → 隐藏，改为页面内路由 // 3. 内容区 → flex:1 占满剩余空间 // 4. 顶部栏 → 保留但简化（logo + 汉堡菜单 + 用户头像）

五、风险评估与保障措施

风险项等级描述保障措施
PC端回归Bug 中 修改共享组件可能影响PC端 媒体查询严格隔离移动端样式；改造前后对比测试；保留原始逻辑分支
复杂组件不适配 高 甘特图/BPMN/日历等手机端无法操作 检测设备类型，移动端显示"请在PC端操作"提示，降级为只读预览
性能问题 低 移动端设备性能较弱 懒加载、代码分割、图片压缩
样式冲突 中 Element Plus 移动端样式覆盖 使用 :deep() 穿透；按需覆盖；移动端样式文件独立管理

六、需要您确认的关键决策

⚠️ 未决问题

1. 哪些页面手机端必须能完整操作？
比如：我的任务、请假申请、待办审批 → 必须能用；甘特图 → 可以只读

2. 手机端是否需要标签页(Tab)机制？
PC端的多标签页在手机上体验差，建议手机端去掉，您同意吗？

3. 底部Tab导航栏放哪几个入口？
建议：首页、我的任务、值班、待办（4个Tab），您觉得呢？

4. 是否需要支持横屏模式？
平板设备旋转屏幕时的适配策略
