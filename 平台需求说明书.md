# Hyper Duty 需求说明书

## 1. 项目概述

### 1.1 项目背景
Hyper Duty是一个企业级的人员管理系统，旨在帮助企业高效管理员工信息、部门结构和用户权限，提供统一的认证授权机制。

### 1.2 项目目标
- 实现人员信息的全生命周期管理
- 支持部门结构的灵活配置
- 提供安全可靠的用户认证授权
- 提供友好的用户界面，便于操作和管理
- 支持系统的可扩展性和可维护性

### 1.3 适用范围
本系统适用于各类企业、组织的人员管理需求，特别是需要对员工信息、部门结构和用户权限进行集中管理的场景。

## 2. 功能需求

### 2.1 认证授权模块

#### 2.1.1 用户登录
- 功能描述：用户通过用户名和密码登录系统，支持邮箱验证码二次验证
- 输入：用户名、密码、验证码（可选）
- 输出：登录成功/失败信息，JWT令牌，员工ID，是否需要验证码标识
- 业务流程：
  1. 用户输入用户名和密码
  2. 系统验证用户名和密码
  3. 检查是否启用了邮箱验证码登录
  4. 如果启用且未提供验证码：返回需要验证码标识，发送验证码到用户邮箱
  5. 如果启用且提供验证码：验证验证码有效性
  6. 验证通过后生成JWT令牌
  7. 返回令牌、用户信息和员工ID

#### 2.1.2 邮箱验证码发送
- 功能描述：向用户邮箱发送登录验证码
- 输入：用户名
- 输出：发送成功/失败信息，脱敏邮箱地址
- 约束：验证码有效期可配置（默认5分钟），发送频率可限制（默认60秒内不能重复发送）

#### 2.1.3 邮件服务配置
- 功能描述：配置邮件服务器参数，用于发送各类通知邮件
- 输入：SMTP服务器地址、端口、是否启用SSL/TLS、发件人邮箱、授权密码、发件人名称、邮件模板、是否启用邮箱验证码登录、验证码有效期
- 输出：保存成功/失败信息
- 功能特性：
  - 支持邮件服务器连接测试
  - 支持多种邮件模板配置（登录验证码、密码找回、异地登录提醒等）
  - 支持邮箱验证码登录开关
  - 支持验证码有效期配置

#### 2.1.4 用户登出
- 功能描述：用户退出系统
- 输入：无（依赖登录状态）
- 输出：登出成功信息
- 业务流程：
  1. 用户点击登出按钮
  2. 系统清除当前用户的认证信息
  3. 返回登出成功信息

##### 2.8.5.1 日报生成
- 功能描述：基于项目任务数据，使用AI智能生成日报
- 输入：报告日期、项目ID（可选）、配置ID（可选）

- 输入：开始日期、结束日期、项目ID（可选）、配置ID（可选）
- 输出：生成确认信息，报告内容异步生成
- 业务规则：
  - 按GOC调度、架构治理、入网管理、变更管控、故障治理、运营治理、数智化研运七大维度分类
  - 将重点任务和非重点任务分开展示
  - 使用JSON格式的结构化数据传递给AI，提升AI理解能力
  - 使用异步处理，避免前端超时
  - AI生成失败时自动重试（默认3次），每次重试会更新提示词并包含错误信息
  - 如果所有重试都失败，使用原始数据生成降级报告
  - 生成的报告保存到数据库，支持后续查看

##### 2.8.5.3 报告历史查询
- 功能描述：查询历史生成的AI报告
- 输入：报告类型（日报/周报，可选）、页码、每页数量
- 输出：报告列表，包括报告ID、报告名称、报告类型、报告日期、创建时间、报告内容等
- 业务规则：
  - 支持按报告类型筛选
  - 支持分页查询
  - 显示报告生成状态（生成中/已完成/失败）

##### 2.8.5.4 报告配置管理
- 功能描述：管理AI报告的配置信息，包括提示词模板、AI模型参数等
- 输入：配置名称、配置编码、报告类型、提示词模板、模型名称、温度、最大Token、Top-P、最大重试次数、System Prompt、状态、备注
- 输出：配置保存成功/失败信息
- 业务规则：
  - 支持配置多个报告模板
  - 支持启用/禁用配置
  - 提示词模板支持变量替换
  - 温度参数：控制AI输出的随机性，取值范围0-1，默认0.7
  - 最大Token：限制AI输出的长度，默认4000
  - Top-P：核采样参数，取值范围0-1，默认0.9
  - 最大重试次数：AI生成失败时的最大重试次数，默认3
  - System Prompt：定义AI的角色定位和行为准则

## 3. 非功能需求

### 3.1 性能需求
- 系统响应时间：页面加载时间≤2秒，操作响应时间≤1秒
- 系统并发能力：支持1000+并发用户
- 数据查询性能：单表查询响应时间≤0.5秒
- 缓存性能：热点数据缓存响应时间≤0.1秒，缓存命中率≥90%

### 3.6 缓存需求
- 缓存策略：使用Redis进行热点数据缓存
- 缓存数据：字典数据、菜单数据、角色数据、用户数据、节假日数据等
- 缓存一致性：确保数据增删改后缓存及时更新
- 缓存过期：设置合理的缓存过期时间，避免缓存膨胀
- 序列化支持：支持Java 8日期时间类型和复杂对象的序列化和反序列化

### 3.2 安全性需求
